## Q&A
### 背景意义类
#### 1 请描述一下国内外目前对于这个问题的研究进展。
答：通过创新性的构思和系统性探索来解决科学问题，是科学发现和研究以及计算机辅助研究的主要目的，这些研究可以自主进行，也可以以辅助人类研究员的方式进行。
创新思想的产生是其中一个重要组成部分。
（具体来说，创新思想的产生可以定义为构建新颖且可行的技术路径或解决方案的过程。）
而目前国内外研究方法主要是两类：一种是传统的**基于规则的方法**，构建领域知识库，设定组合逻辑，来生成方案，比如TRIZ理论、早期专家系统等。
这种方法虽然能够确保逻辑的一致性但在新颖性、泛化能力上很有限；
另一种是**自回归的文本生成方法**，利用大语言模型生成连贯的创意方案，但这种方法本质仍是训练数据中高频模式的重组，可解释性不足容易出现“幻觉”。
这两种方法都不能模拟生成创新思想的内部逻辑结构，这种缺陷使评估新颖性、有效性等维度变得比较困难。
我们的研究将方案的内在逻辑表述为结构化的技术语义图TSG，TSG由技术要素及其逻辑关系构成，充当结构化框架来确保逻辑的完整性，采用迭代优化的工作流，在生产和评估之间交替进行，来实现TSG和文本描述的联合优化，提升其新颖性和可行性。
#### 2 你做研究的创新点在哪里？
我们的核心创新点是使用TSG这个技术结构化的语义图来指导生成过程，并在迭代优化的过程中动态更新TSG和方案描述。整个方法了模仿人类专家的思维过程“框架构建->逻辑验证->迭代优化”，将想法生成从顺序文本提升到了结构化认知建模。

#### 最后方案的评估方法？
评估仲裁员：我们采用一个强大的闭源模型（qwen-max-latest）作为公正的仲裁员，以保证评估结果的质量和一致性
鲁棒性度量：为了增强评估的稳定性和可靠性，我们对每个候选解的每个维度进行三次独立评估，并将平均得分作为最终结果报告。
独立交叉比较：为了确保公平性，在评估特定维度（例如，“新颖性”）时，会同时向评审者展示所有方法的解决方案，并指示其仅根据该维度的标准进行逐一比较评分。此过程独立地针对每个维度重复进行，以防止跨维度评分偏差。
条件评估说明是我们协议的核心（Szymanski et al. 2025）。在评估“结构健全性与可行性”维度时，审裁员将收到以下具体说明：
四个维度：
新颖性（Nov.）：评估解决方案与现有SOTA技术相比的创新程度
有效性
结构健全性与可行性 (SSF)：对解决方案的内部逻辑一致性及其技术可实现性进行综合评估。
问题契合度 (PF)：评估解决方案在多大程度上契合并满足原始问题约束和需求的全部范围。
技术严谨性 (TR)：考察解决方案背后技术论证的深度和可靠性，以及对潜在权衡的考量。
