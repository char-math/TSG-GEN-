# TSG-GEN 笔记
**技术语义图增强的创新思维生成**
## 名词解释
#### 1 自动化科学发现
- **知识驱动发现** ：预先构建静态知识图谱，动态逻辑困难
- **强化学习辅助优化**： 
1）RL 的目标是最大化一个数值奖励信号，这通常与最终的性能指标直接相关。它通过迭代优化来收敛到最佳性能的策略。
2）可能缺乏可解释的内部逻辑结构
- **大型语言模型驱动的生成式辅助**：该方法使用纯文本输出，使得深入验证变得困难

## Q&A
### 背景意义类
#### 1 请描述一下国内外目前对于这个问题的研究进展。
答：通过创新性的构思和系统性探索来解决科学问题，是科学发现和研究以及计算机辅助研究的主要目的，这些研究可以自主进行，也可以以辅助人类研究员的方式进行。
自动化生成创新思想是一个重要组成部分。\
（具体来说，创新思想的产生可以定义为构建新颖且可行的技术路径或解决方案的过程。）\
而目前国内外研究方法主要是两类：一种是传统的**基于规则的方法**，构建领域知识库，设定组合逻辑，来生成方案，比如TRIZ理论、早期专家系统等。
这种方法虽然能够确保逻辑的一致性但在新颖性、泛化能力上很有限；\
另一种是**自回归的文本生成方法**，利用大语言模型生成连贯的创意方案，但这种方法本质仍是训练数据中高频模式的重组，可解释性不足容易出现“幻觉”。\
这两种方法都不能模拟生成创新思想的内部逻辑结构，这种缺陷使评估新颖性、有效性等维度变得比较困难。\
我们的研究将方案的内在逻辑表述为结构化的技术语义图TSG，TSG由技术要素及其逻辑关系构成，充当结构化框架来确保逻辑的完整性，采用迭代优化的工作流，在生产和评估之间交替进行，来实现TSG和文本描述的联合优化，提升其新颖性和可行性。
#### 2 你做研究的创新点在哪里？
我们的核心创新点是使用TSG这个技术结构化的语义图来指导生成过程，并在迭代优化的过程中动态更新TSG和方案描述。整个方法了模仿人类专家的思维过程“框架构建->逻辑验证->迭代优化”，将想法生成从顺序文本提升到了结构化认知建模。

### 方法内容类
#### 1 请介绍一下整篇文章的研究思路。
（1）任务目标是基于一个研究问题描述，构建一个新颖、可行且满足给定需求集的技术解决方案。\
输入问题的自然语言字符串，输出包含一个完整解决方案的二元组（结构化表示，文本描述）\
（2）方法设计。我们提出了TSG-GEN框架，包含四个阶段：\
初始生成：用LLM生成10个候选方案（每个包含TSG和描述）\
精英筛选：用轻量级（3维度）评估筛选出Top-3进入精英池\
迭代优化：对每个精英候选进行最多3轮「评估→反馈→改进」循环，其中评估使用完整（5维度）的多智能体委员会\
最优选择：从优化后的精英池中选出得分最高的方案\
（3）实验验证。我们构建了覆盖8个AI子领域的500问题测试集，设计了直接生成和自我修正两个基线，并通过主实验、消融实验和人工评估三种方式验证了方法的有效性。\
| 实验类型 | 核心发现 | 关键数据 |
| :--- | :--- | :--- |
| 主实验 | TSG-GEN在所有维度上显著优于基线 | 平均分9.09 vs 直接生成6.67（提升36%） |
| 消融实验 | TSG结构是贡献最大的单一模块 | 去掉TSG后性能9.09->7.46下降1.63分（降幅18%） |
| 人工评估 | LLM评分与人类专家判断高度一致 | TSG-GEN在81%的情况下被判定优于基线 |
#### 2 实证研究是如何进行的？
（1）数据集构建。我们构建了500个问题的测试集，覆盖8个AI子领域，包括机器学习、计算机视觉、自然语言处理、物理与工程、优化与运筹学、数据科学、医学影像和人文计算。每个问题都来自真实科研场景或基于论文/专利改写。\
（2）基线设置。我们对比了两个基线：\
Direct-Gen：直接让LLM一次生成方案，无迭代\
Self-Correction：生成→自评→改进，循环3轮，但不使用TSG结构\
（3）评估指标。我们设计了5个评估维度：新颖性、有效性、可行性、问题匹配度、技术严谨性。由5个扮演不同专家角色的LLM智能体独立打分，最终得分为五维平均。\
（4）实验流程。每个问题独立运行完整流程，记录最终得分和各维度得分。我们还做了消融实验——分别去掉TSG、换成单评估器、去掉精英池、去掉迭代——来量化每个模块的贡献。
#### 3 论文中所用的数据是来自于哪里？
我们编制了一个包含500个问题来自人工智能相关领域的论文/专利\
为确保测试的广度和代表性，涵盖了8个主流人工智能研究方向：\
物理与工程（P&E）、自然语言处理与语音（NLP&S）、数据科学（DS）、计算机视觉与图像处理（CV&IP）、医学影像（MI）、机器学习与人工智能（ML&AI）、人文科学（计算）（Hum.）以及优化与运筹学（O&OR）。\
其中，ML&AI、CV&IP和NLP&S是代表性最高的三个领域。\
问题1：如何充分利用偏振信息来增强SAR船舶分类？（P&E）\
问题2：如何利用图像信息来提高三维人体姿态估计的准确性？（CV&IP）\
问题3：如何加速控制变量遗传编程的实验调度？（运筹与管理科学）\
问题4：如何改进离线策略的平稳分布校正估计？（机器学习与人工智能）

### 展望未来类
#### 1 研究不足有哪些？
(1) 对于模糊问题的处理能力有限。当输入问题过于抽象、缺乏明确技术约束时（如「如何让AI更有创造力」），TSG容易生成重复或冗余的节点，方案质量下降明显。\
(2) 评估依赖LLM。虽然我们使用了多智能体评估委员会来降低偏见，并在论文中通过人工盲测验证了LLM评分的相对可靠性，但评分本质上仍是「大模型评大模型」，绝对分数的意义需要谨慎解释。
#### 2 如果继续研究可以从哪些方面改进？
(1)引入检索增强生成（RAG）。当前TSG完全依赖LLM的内部知识，无法利用最新发表的论文或专利。可以构建技术文献向量库，在生成TSG前检索相关文献，将检索结果作为上下文注入提示词，提升方案的信息密度和新颖性。\
(2)构建混合评估体系。将当前的纯LLM评估，扩展为「LLM评估 + 规则检查 + 人工抽样」的混合体系。例如，用规则检查TSG中是否存在孤立节点或循环依赖，用人工评估验证LLM评分的可靠性。

#### 最后方案的评估方法？
评估仲裁员：我们采用一个强大的闭源模型（qwen-max-latest）作为公正的仲裁员，以保证评估结果的质量和一致性
鲁棒性度量：为了增强评估的稳定性和可靠性，我们对每个候选解的每个维度进行三次独立评估，并将平均得分作为最终结果报告。
独立交叉比较：为了确保公平性，在评估特定维度（例如，“新颖性”）时，会同时向评审者展示所有方法的解决方案，并指示其仅根据该维度的标准进行逐一比较评分。此过程独立地针对每个维度重复进行，以防止跨维度评分偏差。
条件评估说明是我们协议的核心（Szymanski et al. 2025）。在评估“结构健全性与可行性”维度时，审裁员将收到以下具体说明：
四个维度：
新颖性（Nov.）：评估解决方案与现有SOTA技术相比的创新程度
有效性
结构健全性与可行性 (SSF)：对解决方案的内部逻辑一致性及其技术可实现性进行综合评估。
问题契合度 (PF)：评估解决方案在多大程度上契合并满足原始问题约束和需求的全部范围。
技术严谨性 (TR)：考察解决方案背后技术论证的深度和可靠性，以及对潜在权衡的考量。
